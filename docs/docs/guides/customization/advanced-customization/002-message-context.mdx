# Interactive Agent Responses

The best part of being able to use response templates and rendering the agent response as a rich user interface is that it allows for an _interactive_ experience. For example,
instead of making the user type out long responses, they can conveniently fill out a form or press buttons in the agent response. This can be achieved by using **message context**.

Message context is a tool that allows you to access the [`Message`](../../../reference/js/react-core/type-aliases/Message.md) object in your response template components.
This is useful if you want to modify or format the message before rendering it, or if you want to implement persistent forms in your response template.

Here's an example demonstrating how to implement a form that collects the user's name and email address, and persist it using message context.

<Steps>

<Step title="Create a form component">

First, create a form component that will be used to render the response template to collect the user's name and email address.

<Tabs groupId="frontend-lang">

<TabItem value="ts" label="TypeScript">

```tsx title="Form.tsx"
import { useState } from "react";
import { useThreadActions } from "@crayonai/react-core";

export const Form = () => {
  const { processMessage } = useThreadActions();

  const [name, setName] = useState("");
  const [email, setEmail] = useState("");

  const handleSubmit: React.FormEventHandler<HTMLFormElement> = async (e) => {
    e.preventDefault();
    await processMessage({
      role: "user",
      message: `Name: ${name}, Email: ${email}`,
      type: "prompt",
      isVisuallyHidden: true, // This will hide the message from the user but still send it to the LLM
    });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Name"
      />
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
      />
      <button type="submit">Submit</button>
    </form>
  );
};
```

</TabItem>

<Step title="Use message context to persist the form data">

Now the form is functional, but it would be odd if the user filled and submitted the form, but the form appeared empty the next time they reloaded the page. To prefill the
form with the previously submitted data, you can use message context to access the `Message` object and subsequently use this object to update the `AssistantMessage` metadata.

This can be done by making the following modifications to the form component:

<Tabs groupId="frontend-lang">

<TabItem value="ts" label="TypeScript">

```tsx title="Form.tsx"
import { useState } from "react";
import { useThreadActions, useMessage } from "@crayonai/react-core";

export const Form = () => {
  // highlight-next-line
  const { message } = useMessage(); // access the message using message context
  const { processMessage, updateMessage } = useThreadActions();

  const [name, setName] = useState<string>(
    // highlight-next-line
    (message.context?.[0]?.name as string) ?? "" // prefill if the message context contains a name
  );
  const [email, setEmail] = useState<string>(
    // highlight-next-line
    (message.context?.[0]?.email as string) ?? "" // prefill if the message context contains an email
  );

  const handleSubmit: React.FormEventHandler<HTMLFormElement> = async (e) => {
    e.preventDefault();
    await processMessage({
      role: "user",
      message: `Name: ${name}, Email: ${email}`,
      type: "prompt",
      isVisuallyHidden: true, // This will hide the message from the user but still send it to the LLM
    });

    // highlight-start
    // store the newly submitted data in the message context
    message.context = [{ name, email }];
    // update the assistant message metadata to contain the newly submitted data
    updateMessage(message, true);
    // highlight-end
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Name"
      />
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
      />
      <button type="submit">Submit</button>
    </form>
  );
};
```

:::note
Message context does not refer to `message.context`, which is the metadata associated with the `Message` object. Instead, it refers to the actual
<ins>[`MessageContext`](../../../reference/js/react-core/functions/MessageContext.md)</ins> that allows you to access the `Message` object.
